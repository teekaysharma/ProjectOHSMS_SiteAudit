Comprehensive guide for Occupational Health and Safety audit professionals

System Overview

What is OHS Management System?

The OHS Management System v2.5 is a comprehensive web-based application designed for trained Occupational Health and Safety professionals to conduct systematic safety audits, track performance metrics, and generate professional reports for compliance and continuous improvement.

Important: This tool is designed exclusively for trained Health and Safety professionals who understand audit methodologies, safety regulations, and risk assessment procedures.

Key Features

📊 Executive Dashboard

Real-time performance metrics with interactive charts and comprehensive analytics for management oversight.

🏢 Management System Audit

Comprehensive project-wide audit covering policy, responsibilities, and documentation compliance.

🏭 Site Performance Audit

Detailed site-specific audits covering permits, safety measures, and operational compliance.

📈 Advanced Analytics

Multi-dimensional analysis with interactive charts, performance comparisons, and trend identification.

📋 Professional Reporting

Customizable report generation with multiple output formats and professional branding options.

🔧 Project Management

Multi-project support with site management, templates, and configuration export/import.

System Architecture

The system is built on modern web technologies to provide a responsive, secure, and user-friendly experience:

The OHS Management System uses a client-side architecture with modern web technologies, ensuring fast performance and offline capability.

Frontend Technologies:

  • HTML5, CSS3, and JavaScript (ES6+)
  • Chart.js 3.7.1 for interactive visualizations
  • Responsive design for mobile and desktop
  • Modern UI with CSS Grid and Flexbox

Development Environment:

  • Vite 5.4.2 for fast development and building
  • ES6 modules for clean code organization
  • No server-side dependencies required
  • Runs entirely in the browser

Data Storage:

  • Browser localStorage for session persistence
  • JSON format for structured data storage
  • Export/Import functionality for backup
  • No external database required

Scoring System

Professional Judgment Required: Scoring should be based on professional expertise and knowledge of applicable regulations.
Score Meaning Description
0 Not Applicable/Not Observed Item does not apply to the audit scope or was not observed during assessment
1 Major Non-Conformance Critical safety issue requiring immediate corrective action
2 Minor Non-Conformance Issue requiring corrective action but not immediately critical
3 Observation/Improvement Area identified for improvement or best practice opportunity
4 Conformance Meets requirements and standards
5 Best Practice Exceeds requirements and demonstrates excellence
Performance Rating Calculation:
  • Percentage = (Average Score / 5) × 100
  • Exclude score 0 (Not Applicable) from calculations
  • Rating based on percentage: Excellent (>90%), Good (>80%), Satisfactory (>70%), Low (>50%), Unacceptable (≤50%)

Count of completed audit items versus total items, showing audit progress and completion status.

Total count of Major and Minor non-conformances identified during audits.

🎯 Best Practices

Number of areas identified as Best Practices, showing excellence in safety management.

📈 Trend Analysis

Performance trends over time, showing improvement or deterioration in safety performance.

The Management System Audit evaluates the project-wide occupational health and safety management system, focusing on policy, responsibilities, planning, and documentation. This audit assesses the foundation of your safety management framework.

Professional Expertise Required: This audit should be conducted by trained OHS professionals with experience in safety management systems and regulatory compliance.

Audit Sections

📋 Health and Safety Policy

Evaluates the existence, adequacy, and implementation of the organization's health and safety policy.

👥 Project Responsibilities

Assesses the assignment of health and safety responsibilities and accountability throughout the project.

📝 Safety Plans

Reviews the development, implementation, and effectiveness of project health and safety plans.

    Examine health and safety policy documents

    Review responsibility matrices and organizational charts

    Analyze safety plans and procedures

  2. Interview Personnel

    Discuss policy understanding with management

    Verify responsibility awareness with team members

    Assess implementation knowledge

  3. Observe Implementation

    Monitor policy communication and display

    Observe responsibility execution

    Verify plan implementation in daily operations

  4. Complete Assessment

    Navigate to Management System tab

    Score each audit element (0-5 scale)

    Add detailed comments and observations

Scoring Guidelines

Score Policy Criteria Responsibility Criteria Planning Criteria

4 Policy meets all requirements Responsibilities clearly defined and understood Adequate plans covering all aspects

    Data Management and Security

    Data Storage Overview

    The OHS Management System stores all data locally in the browser using localStorage, ensuring data privacy and enabling offline access. All data remains on the user's device and is not transmitted to external servers.

    Privacy First: All audit data is stored locally on your device, ensuring complete privacy and control over sensitive information.

    Data Structure

    The system uses a structured JSON format to organize audit data:

    { "masterConfig": { "management": { "Section Name": ["Question 1", "Question 2", ...] }, "site": { "Section Name": ["Question 1", "Question 2", ...] } }, "inspectionData": { "projects": { "Project Name": { "managementSystemAudit": { "Section": [ { "name": "Question", "score": 0-5, "comment": "..." } ] }, "sites": { "Site Name": { "Section": [ { "name": "Question", "score": 0-5, "comment": "..." } ] } }, "currentSite": "Site Name" } }, "currentProject": "Project Name" } }

    Troubleshooting

    Common Issues and Solutions

    Installation and Setup Issues

    Node.js Not Found

    Symptoms: Launcher reports "Node.js is not installed!"

    Solution:

    1. Download Node.js from nodejs.org
    2. Install the LTS (Long Term Support) version
    3. Restart your computer after installation
    4. Run the launcher again
    npm Installation Fails

    Symptoms: Error messages during "npm install" process

    Solutions:

    • Check internet connection
    • Clear npm cache: npm cache clean --force
    • Try different registry: npm install --registry https://registry.npmjs.org/
    • Disable antivirus temporarily during installation
    Port Already in Use

    Symptoms: Server fails to start due to port conflict

    Solution: The system will automatically find an available port. Check the console output for the actual URL.

    Performance Issues

    Slow Loading

    Symptoms: Application takes a long time to load or respond

    Solutions:

    • Close other browser tabs and applications
    • Clear browser cache and cookies
    • Check available system memory
    • Try a different browser
    Charts Not Loading

    Symptoms: Charts appear blank or show loading errors

    Solutions:

    • Check browser console for JavaScript errors
    • Ensure Chart.js library is loaded correctly
    • Verify data integrity in audit records
    • Try refreshing the page

    Data Issues

    Data Not Saving

    Symptoms: Audit data disappears after page refresh

    Solutions:

    • Check browser localStorage permissions
    • Ensure browser is not in private/incognito mode
    • Verify available storage space
    • Try exporting data and re-importing
    Data Corruption

    Symptoms: System shows errors or crashes when accessing data

    Solutions:

    • Clear browser data for the site
    • Restore from backup if available
    • Recreate affected projects or audits
    • Contact support if issue persists

    Browser Compatibility Issues

    Browser Not Supported

    Symptoms: Application doesn't work properly in current browser

    Solution: Use a supported browser:

    • Chrome 90+
    • Firefox 88+
    • Safari 14+
    • Edge 90+
    Mobile Display Issues

    Symptoms: Application doesn't display correctly on mobile devices

    Solutions:

    • Ensure device is in landscape orientation
    • Check browser zoom level (100% recommended)
    • Try a different mobile browser
    • Use desktop version for complex tasks

    Q: What is the OHS Management System?

    Q: Who should use this system?

    A: This system is designed exclusively for trained and qualified Health and Safety professionals who have expertise in audit methodologies, safety regulations, and risk assessment procedures.

    Q: Is this system free to use?

    A: Yes, the system is open-source and free to use under the MIT license. However, users are responsible for ensuring compliance with applicable regulations and standards.

    Q: What are the system requirements?

    A: You need a modern web browser (Chrome 90+, Firefox 88+, Safari 14+, Edge 90+), Node.js 16+, and 4GB of RAM. The system runs entirely in your browser with no server requirements.

    Installation and Setup

    Q: How do I install the system?

    A: Download the ZIP file from GitHub, extract it to a folder, install Node.js if needed, then double-click run.bat (Windows) or run npm install && npm run dev from command line.

    Q: Do I need technical expertise to install this?

    A: The Windows launchers make installation simple with automatic dependency checking and guided setup. However, you should be comfortable with basic computer operations.

    Q: Can I install this on multiple computers?

    A: Yes, you can install the system on multiple computers. Each installation will have its own local data storage.

    Q: Does this require an internet connection?

    A: Internet connection is only required for initial installation and dependency downloads. Once installed, the system works offline.

    Usage and Features

    Q: How do I create my first audit?

    A: Launch the application, click "System Settings" → "Load Default Template", create a new project, add sites if needed, then start completing the audit forms.

    Q: Can I customize the audit questions?

    A: Yes, you can customize questions by going to "System Settings" → "Template Management" and modifying the questions to suit your specific needs.

    Q: How do I generate reports?

    A: Navigate to the "Reports" tab, select the type of report you want, configure the options, and click "Generate Report". You can export as HTML or PDF.

    Q: Can I compare performance across multiple sites?

    A: Yes, use the Analytics tab to select multiple sites and generate comparison charts with stacked bar, grouped bar, or radar chart visualizations.

    Data and Security

    Q: Where is my data stored?

    A: All data is stored locally in your browser's localStorage. No data is transmitted to external servers or third parties.

    Q: How do I backup my data?

    A: Go to "System Settings" → "Data Management" → "Export Project Data" to save your data as a JSON file. Store this file in a secure location.

    Q: Is my data secure?

    A: Data security depends on your device security. Since data is stored locally, you are responsible for device security, backup procedures, and access control.

    Q: Can I share audit data with others?

    A: Yes, you can export project data and share the JSON files, or generate reports in PDF/HTML format for distribution.
